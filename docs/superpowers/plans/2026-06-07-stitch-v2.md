> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox syntax for tracking.

**Validated constants (prototyped 2026-06-07, do not retune blindly):** the v2
algorithm below was prototyped against the exact QPainter fixture in Task 4.
Results: integral offsets → exact offsets, confidence 1.0; fractional (kinetic)
offsets → offsets within ±1, full reconstruction mean-abs-diff 1.58 (bar: < 8.0)
with height exactly 540 under the final reference policy; a 50/50 blend
"mid-animation blur" frame →
`(None, 0.0)` (dropped); unrelated frames → `(None, 0.0)`; blank frame →
`(None, 0.0)`; fixed-header chrome → `static_bands` locks at exactly 30 and the
output is an exact contiguous slice of the page. Constants: `band=64`,
`threshold=8.0`, `n_bands=5`, `band_w=48`, `var_min=100.0`, `tol=2`, consensus
acceptance at `conf >= 0.5`, fallback acceptance at `score < threshold * 0.5`
with confidence `0.5`, stitcher gate `min_confidence=0.6`.

Note (intentional, do not "fix"): the fallback's confidence (0.5) is below the
stitcher's default gate (0.6), so a fallback-accepted offset is still dropped
by `ScrollStitcher` at defaults — that is the seam policy (drop beats jagged).
The fallback exists for `detect_offset` callers with lower gates and for
diagnostics; do not raise its confidence to sneak it past the gate.

**Reference-frame policy change (also prototyped):** v1 resynced `_prev_gray`
to every incoming frame even on a drop ("always resync"). That is a latent
canvas-corruption bug: after a dropped frame, the next frame's offset is
measured against the *dropped* frame but `vstack` assumes the canvas bottom is
the last *stitched* frame — a drop-then-match appends rows with a content gap.
v2 matches against the last stitched frame instead (update the reference gray
only when a frame is used). Prototype: blur-then-clean run recovers (used=3,
height exactly 450, mean diff 0.34) and the kinetic run's accumulated diff
improves (1.58 vs 3.57) because drops no longer shift the reference. Side
effect: after a true scene change every later frame fails to match and is
dropped (truncated output) instead of stitching unrelated content under the old
scene — strictly better under the seam policy. All existing tests still pass
under the new policy (verified by simulation: no-motion drops compare equal to
the last used frame either way).

**Background:** v1 `detect_offset(prev, cur, band=64, threshold=8.0)` slides the
full-width top-64-rows strip of `cur` down `prev`, returns the best integer
offset, or `None` if nothing beats `threshold` mean-abs-diff. On real pages a
single full-width band straddles flat regions and kinetic-scroll smear,
producing wrong offsets → jagged seams. v2 samples up to 5 narrow
(`band_w=48`-column) bands at evenly spaced x positions, keeps only textured
ones (variance ≥ 100), estimates an offset per band, takes the median with ±2
outlier rejection, and reports `confidence` = inlier fraction over candidate
positions. If consensus is weak (< 0.5) it falls back to a full-width
full-overlap slide accepted only on a strong score (< threshold/2) at
confidence 0.5. Return type becomes `(offset | None, confidence)`.

  # -- offset detection (v2: multi-band consensus) ---------------------------

  def _slide_band(prev_col: np.ndarray, strip_col: np.ndarray,
                  threshold: float) -> tuple[int | None, float]:
      """Slide strip_col down prev_col; return (best_d, best_score).

      best_d is None when no candidate's mean abs difference beats
      threshold (in which case best_score is the threshold itself).
      Ties keep the smallest d (strict <).
      """
      h = prev_col.shape[0]
      band = strip_col.shape[0]
      best_d: int | None = None
      best_score = threshold
      for d in range(0, h - band + 1):
          score = float(np.abs(prev_col[d:d + band] - strip_col).mean())
          if score < best_score:
              best_d, best_score = d, score
      return best_d, best_score


  def textured_xs(strip: np.ndarray, band_w: int, n: int,
                  var_min: float) -> list[int]:
      """Up to n evenly spaced x positions whose (band_h, band_w) patch
      has variance >= var_min — flat patches (margins, blank panels)
      match everywhere and must not vote. Narrow images (width <=
      band_w) get the single full-width band at x=0."""
      w = strip.shape[1]
      if w <= band_w:
          return [0]
      xs = np.linspace(0, w - band_w, n).astype(int)
      return [int(x) for x in xs
              if float(strip[:, x:x + band_w].var()) >= var_min]


  def _consensus(ds: list[int], n_candidates: int,
                 tol: int) -> tuple[int | None, float]:
      """Median offset with outlier rejection.

      confidence = inlier fraction over the *candidate* band count, so
      bands that found no match at all (None) also dilute confidence.
      """
      if not ds:
          return None, 0.0
      med = int(np.median(ds))
      inliers = [d for d in ds if abs(d - med) <= tol]
      conf = len(inliers) / max(n_candidates, 1)
      return int(round(float(np.median(inliers)))), conf


  def detect_offset(prev: np.ndarray, cur: np.ndarray,
                    band: int = 64, threshold: float = 8.0,
                    n_bands: int = 5, band_w: int = 48,
                    var_min: float = 100.0, tol: int = 2,
                    ) -> tuple[int | None, float]:
      """Vertical scroll distance between two grayscale frames.

      Takes the top `band` rows of cur and matches them against prev at
      up to `n_bands` texture-selected x positions (variance >= var_min;
      flat bands are skipped — they match everywhere). Each band votes
      an offset; the median with +-tol outlier rejection wins, and
      confidence is the inlier fraction. When consensus is weak
      (< 0.5) a full-width full-overlap slide is tried as a fallback,
      accepted only on a strong score (< threshold/2) at confidence
      0.5. Kinetic scrolling lands frames at fractional offsets: bands
      then disagree by +-1 (absorbed by tol) and mid-animation smear
      pushes per-band scores past threshold, so confidence collapses
      and the caller can skip the frame instead of mis-stitching it.

      Returns (offset, confidence): offset 0 means no motion, d > 0 a
      downward scroll, None no reliable match (scene change, blur,
      featureless frame). confidence is 0.0..1.0.
      """
      h = prev.shape[0]
      band = min(band, h)
      strip = cur[:band]
      xs = textured_xs(strip, band_w, n_bands, var_min)
      votes = [
          d for x in xs
          for d, _ in (_slide_band(prev[:, x:x + band_w],
                                   strip[:, x:x + band_w], threshold),)
          if d is not None
      ]
      d, conf = _consensus(votes, len(xs), tol)
      if d is not None and conf >= 0.5:
          return d, conf
      # Bands disagree or none were textured/matchable: full-overlap
      # correlation across the full width, accepted only when decisive.
      d_full, score = _slide_band(prev, strip, threshold)
      if d_full is not None and score < threshold * 0.5:
          return d_full, 0.5
      return None, 0.0
  ```

**Seam policy:** a longer scroll with more frames beats a jagged seam. Any
frame pair whose match confidence is below `min_confidence` is dropped (counted
in `frames_dropped`). And the match reference becomes the last *stitched*
frame, not the last *seen* frame: v1's "always resync" meant a drop-then-match
appended rows measured against the dropped frame while the canvas bottom was
the last stitched frame — a content gap. With the new policy a clean frame
after a blur frame re-matches against the last stitched frame and the canvas
stays contiguous (prototype: blur run reconstructs exactly 450 rows, mean diff
0.34).

  # -- realistic fixtures: QPainter text-like pages (stitch v2 bar) ----------

  def render_text_page(height=1400, width=420, seed=3):
      """Text-like page: rows of varying-width rounded rects (words),
      margins, occasional paragraph gaps. Returns (H, W, 3) uint8."""
      from PySide6.QtCore import QRectF, Qt
      from PySide6.QtGui import QColor, QImage, QPainter
      from wondershot.stitch import qimage_to_rgb
      rng = np.random.default_rng(seed)
      img = QImage(width, height, QImage.Format_RGB32)
      img.fill(QColor("white"))
      p = QPainter(img)
      p.setRenderHint(QPainter.Antialiasing)
      y = 18
      while y < height - 24:
          x = 28
          while x < width - 80:
              w = int(rng.integers(18, 70))
              shade = int(rng.integers(40, 90))
              p.setBrush(QColor(shade, shade, shade))
              p.setPen(Qt.NoPen)
              p.drawRoundedRect(QRectF(x, y, w, 11), 3, 3)
              x += w + 9
          y += 19 if rng.random() > 0.15 else 34  # paragraph gaps
      p.end()
      return qimage_to_rgb(img)


  def viewport_at(page: np.ndarray, y: float, vh: int) -> np.ndarray:
      """Viewport sampled at a (possibly fractional) offset via linear
      row interpolation — what a compositor mid-smooth-scroll shows.
      QPainter cannot do this: unscaled image blits snap to integer
      positions even with SmoothPixmapTransform (measured)."""
      i = int(np.floor(y))
      f = float(y) - i
      if f == 0.0:
          return page[i:i + vh].copy()
      a = page[i:i + vh].astype(np.float32)
      b = page[i + 1:i + 1 + vh].astype(np.float32)
      return ((1.0 - f) * a + f * b).astype(np.uint8)
  ```
